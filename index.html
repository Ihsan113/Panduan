LANGKAH 1.3B Upload File Project — Via Terminal / Git ▼

🔑 Penting! GitHub Sekarang Pakai Token, Bukan Password

Kalau kamu push lewat terminal, GitHub tidak terima password biasa lagi. Kamu harus buat Personal Access Token (Classic) dulu sebagai pengganti password.

  • Login ke GitHub, klik foto profil → Settings
  • Scroll ke bawah, klik Developer settings (menu paling bawah kiri)
  • Klik Personal access tokens → Tokens (classic)
  • Klik Generate new token → Generate new token (classic)
  • Isi Note: misal topup-deploy
  • Set Expiration: bebas (misal 90 days atau No expiration)
  • Centang scope: repo (cukup ini saja)
  • Klik Generate token di bawah
  • SALIN tokennya sekarang! Token hanya tampil sekali. Simpan di notepad.
https://github.com/settings/tokens
💡 Token ini yang kamu paste sebagai password saat terminal minta password. Username tetap username GitHub-mu.

Setelah punya token, jalankan perintah ini:

# Masuk ke folder project cd topup-fixed # Inisialisasi git git init git add . git commit -m "first commit" # Hubungkan ke GitHub (ganti username & nama repo) git remote add origin https://github.com/username/topup-store.git git branch -M main git push -u origin main # Saat diminta: # Username: username GitHub-mu # Password: paste TOKEN yang tadi disalin (bukan password GitHub!)
💡 Kalau mau biar tidak diminta token terus setiap push, jalankan: git config --global credential.helper store — token akan disimpan otomatis.

Troubleshooting

Solusi kalau ada masalah — klik untuk lihat

🚫 Website tidak bisa dibuka / error 404 ▼
  • Pastikan file vercel.json ikut terupload ke GitHub
  • Di Vercel → Settings → General, pastikan Framework Preset: Vite
💳 Pembayaran berhasil tapi transaksi tidak diproses ▼
  • Cek apakah Webhook URL sudah diisi di Flowix
  • Cek apakah WEBHOOK_SECRET di Vercel sama dengan di dashboard Flowix
  • Lihat log error: Vercel → Functions → /api/webhook → Logs
🗄️ Database error / tidak bisa simpan data ▼
  • Pastikan MONGODB_URI benar — tidak ada spasi atau karakter salah
  • Cek Network Access di MongoDB Atlas → sudah allow 0.0.0.0/0?
  • Pastikan user database sudah dibuat dan passwordnya benar
🔐 Tidak bisa login ke /admin ▼
  • Password admin diambil dari variabel VITE_ADMIN_PASSWORD di Vercel
  • Kalau baru ubah nilainya, lakukan Redeploy terlebih dulu
🔑 Git push ditolak / authentication failed ▼
  • GitHub tidak lagi menerima password biasa untuk push via terminal
  • Kamu harus pakai Personal Access Token (Classic) — lihat Langkah 1.3B
  • Buat token di: github.com/settings/tokens
  • Saat diminta password di terminal, paste token tersebut — bukan password GitHub-mu

Auto Push GitHub via Termux (HP Android)

Upload project dari HP ke GitHub tanpa laptop — pakai script Python

📱 Pastikan: Download Termux dari F-Droid (bukan Play Store yang sudah outdated). Simpan semua folder project di folder Download HP kamu.
📂 Sebelum mulai, buka Pengaturan HP → Info Aplikasi → Termux → Izin → Penyimpanan, pastikan izin penyimpanan sudah Diizinkan. Atau izinkan saat muncul pop-up di langkah pertama.
LANGKAH 9.1 Install Lingkungan (Git, Python & Akses Storage) ▼

Jalankan perintah berikut satu per satu di Termux:

# Update sistem dulu pkg update && pkg upgrade -y # Install Git dan Python pkg install git python -y # Beri izin akses ke folder HP (SD Card/Internal) termux-setup-storage # → Klik "Allow / Izinkan" saat pop-up muncul
💡 Jika pop-up tidak muncul, izinkan manual: Pengaturan → Info Aplikasi → Termux → Izin → Penyimpanan → Izinkan
LANGKAH 9.2 Buat GitHub Personal Access Token (Classic) ▼

GitHub tidak menerima password biasa. Kamu perlu Token (Classic) sebagai pengganti password:

🔑 Cara Buat Token Classic

  • Buka github.com/settings/tokens
  • Klik Tokens (classic) → Generate new token (classic)
  • Note: isi Termux-Push
  • Expiration: pilih No expiration (agar tidak hangus)
  • Centang scope: repo (ini saja sudah cukup)
  • Klik Generate token → SALIN & SIMPAN tokennya sekarang! Token hanya tampil sekali!
⚠️ Token ini yang kamu paste sebagai password saat terminal minta password. Username tetap username GitHub biasa.
LANGKAH 9.3 Siapkan Script Python (autopush.py) ▼

Script autopush.py sudah tersedia di dalam ZIP source code. Tinggal pindahkan ke folder Download:

1
Ekstrak ZIP source code → cari file autopush.py
2
Pindahkan file autopush.py ke folder Download HP kamu
3
Di Termux, masuk ke folder Download:
cd /sdcard/Download # atau cd ~/storage/downloads

Simpan token agar tidak perlu dimasukkan ulang tiap push:

git config --global credential.helper store
💡 Tidak perlu nano lagi karena script sudah ada di ZIP. Cukup pindah ke folder Download lalu jalankan.
LANGKAH 9.4 Jalankan Script & Cara Pengisian ▼

Masuk ke folder yang berisi script, lalu jalankan:

cd /sdcard/Download python autopush.py

Isi input saat script berjalan:

Input yang DitanyaContoh Isian
Nama & EmailIsi sesuai akun GitHub kamu (Nama / email@gmail.com)
Lokasi Folder ProjectDi Download HP: /sdcard/Download/topup-fixed
URL Repositoryhttps://github.com/username/topup-store.git
Pesan CommitBebas, contoh: Update fitur baru
Username GitHubKetik username GitHub kamu
PasswordPASTE TOKEN yang dibuat di Langkah 9.2 (karakter tidak terlihat, langsung Enter)
✅ Setelah berhasil, cek repo GitHub kamu — file-file project sudah terupload!

Deploy Langsung dari Termux ke Vercel (Tanpa GitHub)

Alternatif deploy via Vercel CLI — upload langsung dari HP, tanpa perlu repo GitHub

💡 Metode ini cocok kalau kamu tidak ingin pakai GitHub sama sekali. Vercel CLI bisa deploy langsung dari folder lokal HP ke Vercel, tanpa perlu push ke repo dulu.
📱 Pastikan Termux sudah ter-install dari F-Droid dan sudah punya akses penyimpanan (termux-setup-storage). Lihat Bagian 9 jika belum setup.
LANGKAH 11.1 Install Node.js & Vercel CLI di Termux ▼

Jalankan perintah berikut satu per satu di Termux:

# Update paket Termux dulu pkg update && pkg upgrade -y # Install Node.js (sudah include npm) pkg install nodejs -y # Verifikasi versi Node & npm node -v npm -v # Install Vercel CLI secara global npm install -g vercel # Verifikasi Vercel CLI berhasil ter-install vercel --version
✅ Jika muncul versi Vercel (contoh: Vercel CLI 40.x.x), berarti install berhasil dan siap dipakai.
LANGKAH 11.2 Login ke Akun Vercel dari Termux ▼

Login ke akun Vercel-mu langsung dari terminal:

vercel login

Setelah perintah dijalankan:

1
Terminal akan menampilkan beberapa opsi login — pilih Continue with Email
2
Masukkan email yang kamu pakai di Vercel, lalu tekan Enter
3
Buka email kamu → cari email dari Vercel → klik tombol Verify
4
Kembali ke Termux — akan muncul pesan "Congratulations! You are now logged in"
💡 Alternatif: pilih Continue with GitHub jika kamu sudah punya akun GitHub yang terhubung ke Vercel. Browser akan terbuka otomatis untuk otorisasi.
LANGKAH 11.3 Masuk ke Folder Project & Jalankan Deploy ▼

Pindah ke folder project kamu di HP, lalu deploy langsung:

# Masuk ke folder project (sesuaikan nama folder) cd /sdcard/Download/topup-fixed # Jalankan deploy ke Vercel vercel --prod

Jawab pertanyaan yang muncul di terminal:

PertanyaanJawaban
Set up and deploy?Ketik Y → Enter
Which scope?Pilih akun kamu → Enter
Link to existing project?N jika deploy pertama kali → Enter
Project name?Isi nama project (contoh: topup-store) → Enter
In which directory is your code?Biarkan kosong (sudah di folder yang benar) → Enter
Want to modify settings?Ketik N → Enter
⚠️ Jika project sudah pernah di-deploy sebelumnya, jawab Y pada pertanyaan "Link to existing project?" dan pilih nama project yang sesuai.
✅ Setelah proses selesai, Vercel langsung memberikan URL production seperti topup-store.vercel.app. Buka URL tersebut untuk mengecek hasilnya!
LANGKAH 11.4 Isi Environment Variables (Wajib Setelah Deploy Pertama) ▼

Environment Variables tidak bisa diisi lewat CLI saat deploy pertama. Isi via dashboard Vercel setelah project terbuat:

1
Buka vercel.com/dashboard di browser HP
2
Klik nama project yang baru di-deploy → tab Settings → Environment Variables
3
Isi semua variabel sesuai Bagian 5 (termasuk LICENSE_KEY dan LICENSE_SERVER dari admin)
4
Setelah semua terisi, kembali ke Termux dan jalankan ulang deploy:
# Redeploy setelah environment variables diisi cd /sdcard/Download/topup-fixed vercel --prod
⛔ Jangan lupa redeploy setelah mengisi Environment Variables — perubahan tidak berlaku sebelum deploy ulang!
TIPS 🔄 Update Project — Cara Deploy Ulang Setelah Edit File ▼

Setiap kali kamu mengubah file di folder project, tinggal jalankan satu perintah ini dari Termux:

cd /sdcard/Download/topup-fixed vercel --prod

Vercel CLI akan langsung upload perubahan dan memperbarui deployment production-mu. Tidak perlu GitHub sama sekali!
